# v0.3 — Visual Polish

> Goal: make the game visually engaging enough to enjoy playtesting.
> No new mechanics. Engine untouched. UI only.

---

## Target Areas

### 1. Cards
- Proper card shape — fixed aspect ratio, rounded corners, border that reflects the card
- Color-coded by stat profile (e.g. high ATK = warm red tone, high HP = cool blue tone)
- Clear visual hierarchy: name at top, ATK/HP stats prominent, cost in a badge corner
- Selected state is obvious (glowing border, slight lift effect)
- Unaffordable cards are clearly dimmed, not just slightly transparent

### 2. Board & Lanes
- Lanes feel like actual slots — inset/sunken look when empty, raised when occupied
- Frontline and backline rows are visually distinct (e.g. frontline brighter, backline darker/recessed)
- Protected backline lanes communicate "shielded" clearly, not just dimmed
- Active player's side has a subtle highlight so it's always obvious whose turn it is
- Divider between the two players is more prominent — a clear battlefield line

### 3. Player Info Bar
- Core HP shown as a health bar, not just a number
- Energy shown as pips (filled/empty dots up to 6) instead of a raw number
- Player label is clear and prominent

### 4. Phase & Turn Header
- Current phase displayed as a styled badge (MAIN / COMBAT)
- Active player indicator is hard to miss
- Status line (card selected / attacker selected) is readable at a glance

### 5. Action Log
- Alternating row shading so lines are easier to scan
- Color-coded entries: damage = red tint, destruction = orange, turn change = blue, win = gold

### 6. Controls / Buttons
- Buttons styled to match the theme — not default browser grey
- Enter Combat button looks aggressive (warm color)
- End Turn button looks conclusive (neutral/cool)
- Clear hover and active states

---

## Constraints
- No external UI libraries (same as before)
- Engine files untouched
- All styling via inline styles + styles.css
- No animations for now — save those for a later pass
