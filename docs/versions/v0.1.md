## Board
- 5 frontline lanes per player (no backline)
- Each lane holds at most 1 unit
- Each player has a Core (15 HP)
- Win condition: opponent Core HP ≤ 0

## Turn Structure (Yu-Gi-Oh style)
1. **Start Phase** — draw 1 card
2. **Resource Phase** — gain +2 Energy (cap 6)
3. **Main Phase** — play units from hand into empty lanes
4. **Combat Phase** — active player attacks with units
   - Each unit attacks at most once per turn
   - Must attack the opposing unit in the same lane if present; otherwise hits Core directly
   - No cross-lane attacks
5. **End Phase** — switch active player

## Cards
- Deck size: 12 (hardcoded)
- Starting hand: 5
- Unit cards only (no spells)

| Name         | Cost | ATK | HP |
|--------------|------|-----|----|
| Vanguard     |  2   |  1  |  4 |
| Striker      |  2   |  2  |  3 |
| Glass Cannon |  3   |  4  |  2 |

- Unit at 0 HP → removed to discard pile

## State Management
- Reducer in `engine/reducer.ts` with typed Actions
- All rules in `engine/rules.ts`
- State fully serializable (no class instances)
- Illegal actions return state unchanged and log an error

## Smoke Tests (all passing)
- Game init: hands, decks, energy
- Playing a unit reduces energy and removes card from hand
- Combat hits unit when present, otherwise hits Core
- Win condition triggers when Core reaches 0

## Out of Scope (deferred)

- Towers
- Backline lanes

- Special keywords
