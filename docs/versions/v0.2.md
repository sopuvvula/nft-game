# v0.2 — Backline

> Goal: introduce the backline row and validate the frontline/backline exposure rules
> without adding any other complexity. One structural change at a time.

---

## What's New

### Backline Lanes
Each player gets a second row of 5 lanes sitting directly behind their frontline.

**Rules:**
- A backline unit is **protected** while a friendly unit occupies the same frontline lane.
  - It cannot be attacked.
  - It cannot attack.
- A backline unit is **exposed** when its frontline lane is empty.
  - It can be attacked directly.
  - It can attack (same lane targeting rules as frontline).
- Units can be played into backline lanes the same way as frontline — click lane, pay cost.

**Board layout:**
```
[ Player B Backline  — 5 lanes ]
[ Player B Frontline — 5 lanes ]
────────────────────────────────
[ Player A Frontline — 5 lanes ]
[ Player A Backline  — 5 lanes ]
```

### New Card Templates
Add 2 backline-oriented cards to the pool to give the row a purpose:

| Name      | Cost | ATK | HP | Notes                  |
|-----------|------|-----|----|------------------------|
| Sentinel  |  3   |  1  |  6 | High HP, sits backline |
| Phantom   |  2   |  3  |  2 | High ATK, fragile      |

Deck size stays at 12. All 5 templates shuffled in (2-3 copies each to hit 12).

## Engine Changes

| File | Change |
|------|--------|
| `types.ts` | Split `PlayerState.lanes` into `frontline` and `backline` (both `(UnitInstance \| null)[]` length 5) |
| `rules.ts` | Add `isExposed(laneIndex, playerState)` — true when frontline lane is empty; update `canAttack` to check exposure; update `canPlayCard` to accept a `row` param |
| `reducer.ts` | `PLAY_CARD` gains a `row: 'frontline' \| 'backline'` field; attack logic reads from correct row |
| `initialState.ts` | Init `frontline` and `backline`; add 2 new card templates |
| `selectors.ts` | Update any selectors that reference `lanes` |
| `tests.smoke.ts` | New tests: backline unit protected when frontline occupied; backline unit exposed and attackable when frontline empty |

## UI Changes

| File | Change |
|------|--------|
| `Board.tsx` | Render 4 rows total (B backline, B frontline, A frontline, A backline); visually dim protected backline lanes |
| `Hand.tsx` | When placing a card, player selects frontline or backline lane (click target row) |

---

## Out of Scope for v0.2
- Spell cards
- Keywords (Rush, Taunt, Shield, etc.)
- Towers
- Cross-lane attacks
- Multiplayer
