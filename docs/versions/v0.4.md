# v0.4 — Unit Keywords

> Goal: add keyword abilities to units, creating real decision-making in deckbuilding and combat.
> First engine-level feature since v0.2. Builds directly on the existing unit system.

---

## Why Keywords First

Keywords are the smallest mechanic that creates the most strategic depth.
They change how existing units behave without adding new card types (spells, towers).
Spells, towers, and field cards all benefit from keywords existing first
(e.g. a spell that grants Shield, a tower that gives adjacent units Taunt).

---

## Keywords (3 total — keep it tight)

### 1. Taunt
- Enemies **must** attack the Taunt unit's lane before other lanes.
- If multiple Taunt units exist, the attacker chooses among them.
- Changes targeting rules — no more freely picking which lane to attack.
- **Use case:** Protect your backline carry by forcing opponents into your tank.

### 2. Shield
- Blocks the **first** instance of damage the unit receives. Then the shield breaks.
- A shielded unit displays a visual indicator.
- Shield does not regenerate.
- **Use case:** Keep fragile high-ATK units alive for one extra hit.

### 3. Piercing
- When this unit kills a frontline/backline unit, **excess damage** carries through to the Core.
- Only the overflow hits Core (e.g. Piercing unit with 4 ATK kills a 2 HP unit → 2 damage to Core).
- **Use case:** Punish low-HP blockers. Reward investing in high-ATK units.

## Engine Changes

### types.ts
- Add `Keyword` type: `'taunt' | 'shield' | 'piercing'`
- Add `keywords: Keyword[]` to `Card` interface
- Add `keywords: Keyword[]` and `shieldActive: boolean` to `UnitInstance`

### initialState.ts
- Add `keywords` field to `CardTemplate` and each template
- Introduce **2 new unit cards** that carry keywords (see below)
- Update deck composition to include new cards

### rules.ts
- `canAttack`: if opponent has any Taunt unit, attacker must target a Taunt lane
- Shield: add `applyDamageToUnit` logic — if `shieldActive`, set to false and skip damage
- Piercing: return overflow damage from `applyDamageToUnit` for reducer to apply to Core

### reducer.ts
- ATTACK case: handle Shield (break instead of damaging), handle Piercing (carry excess to Core)
- Pass keywords from Card → UnitInstance when playing a card

### tests.smoke.ts
- Test: Taunt forces targeting — cannot attack non-Taunt lane when Taunt unit exists
- Test: Shield blocks first hit, second hit deals damage normally
- Test: Piercing carries excess damage to Core

## UI Changes (minimal)
- Show keyword name(s) on the card below stats
- Shield indicator on units (small icon or border glow)
- Taunt indicator on lane (enemy sees "must attack here")
- Log messages reference keywords (e.g. "Shield absorbed the hit!", "Piercing: 2 overflow damage to Core!")

---

## Constraints
- No new card types (spells come later)
- No new phases or turn structure changes
- Keywords only affect units, not the game flow
- Keep total card count manageable (7 templates, 14-card decks)

---

## Out of Scope
- Spell cards
- Towers and field cards
- Keyword combos (e.g. Taunt + Shield on same unit) — allowed if it happens naturally, but not designed around
- Animations
